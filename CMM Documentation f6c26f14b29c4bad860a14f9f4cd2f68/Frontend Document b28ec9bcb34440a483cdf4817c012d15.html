Components

Reusable components

Reusable components can be used throughout the application to avoid code duplication and improve maintainability

  • AlertDialog.jsx - To display alert dialog box after/before every call to action
  • Cards.jsx - To display capabilities in card view for a more understandable view
  • Grid.jsx - Made to reflect the annual growth of a program by heat map
  • Navbar.jsx - To navigate between pages
  • Sidebar.jsx - To navigate between processes
  • Table.jsx - To display the capabilities in table view to give a concise view

Hero Components

Hero components are often used to display prominent content, such as a company’s description or a company logo.

  • Public.jsx - First page of the app. It can also be used to give detail about the CMM
  • Welcome.jsx - First page after the user logs in. Used to select the project name and state to start the goal-setting process

Goal Setting Pages

Goal-setting pages allow users to define and track their goals.

  • AnnualGoalSettingPage.jsx -This component allows users to select goals from a list of cards and save them as a draft or submit them for final approval. It uses the useCapabilityData hook to access the global capability data and selected options from the CapabilityDataProvider context. It also uses the axios library to make HTTP requests to the backend server to save and submit goals.
  • QuarterGoalSettingPage.jsx - This component allows users to set goals for a quarter, based on the annual goals already set. It also uses the useCapabilityData hook to access the global capability data and selected options from the CapabilityDataProvider context. It also uses the axios library to make HTTP requests to the backend server to save the selected goals for the quarter.

Reflection Page

Reflection pages allow users to review their progress

  • QuarterEndRedflectionPage.jsx - This page allows users to view and interact with data related to capabilities, their completion status, and progress over quarters and annually. It provides a graphical representation of this data and allows users to save updates to quarterly goals.

User Setting Pages

User Setting Pages are used to allow users to manage their information and account settings. These pages can include a variety of features, such as the ability to view and edit information and delete an account

  • User.jsx - This component renders a row in a table that displays information about a user, including their username, roles, and an edit button. The component uses the useGetUsersQuery hook from the usersApiSlice to fetch the user data from the backend server. The selectFromResult prop is used to specify that only the user with the specified userId should be returned.
  • EditUser.jsx - This component allows users to edit their user information. The component uses the useParams hook from React Router to get the id of the user to edit. It then uses the useGetUsersQuery hook from the usersApiSlice to fetch the user data from the backend server.
  • NewUserForm.jsx - In this form the new users fill out information to create an account. It would typically ask for basic information such as the user's name, user role, and password.
  • EditUserForm.jsx - This component allows users to edit their user information, including their username, password, roles, and active status. The component uses the useUpdateUserMutation and useDeleteUserMutation hooks from the usersApiSlice to perform the update and delete user actions. It also uses the useNavigate hook from React Router to navigate the user to the /dash/users page after a successful update or deletion.

Context API

Context API is a way to share data between components at different nesting levels without having to pass props down manually.

It consists of two components:

  1. Provider: The Provider component is responsible for providing the data that needs to be shared across the components
  1. Consumer: The Consumer component is responsible for consuming the data provided by the Provider.

For example, the two pages (Annual Goal Setting Page & Quarter Goal Setting Page) use the waterfall model and share the same data of capabilities, it is beneficial to use the Context here.

  • DataContext.jsx - This page is used as the context provider for capability data. The context provider exposes the following values:
    • globalCapabilityData: An array of objects containing the capability data for all programs and states.
    • selectedOptions: An array of objects containing the selected cards.
    • programState: The selected program state.
    • programYear: The selected program year.
    • dataForQuarter: An array of objects containing the capability data for the selected quarter.
    • progress: A number representing the progress of the data fetch.

The context provider also includes two useEffect hooks to fetch the global capability and quarter data for the selected program and state.

Redux components

Redux is a state management library that provides a way to manage global state in React applications.

Redux consists of three parts:

  1. Store: It is a single JavaScript object that holds the state of the application.
  1. Actions: Actions are plain JavaScript objects that describe the changes to the state
  1. Reducers: Reducers are pure functions that take the current state and an action as input and return the next state

  • RequireAuth.jsx - It checks if the user is logged in and has the required roles to access a specific page. It does this by using the useAuth hook to get the user's roles and the useLocation hook to get the current location.
  • authSlice.jsx - Redux slice for authentication. It defines two reducers:
    • setCredentials: This reducer sets the user's access token in the Redux store.
    • logOut: This reducer clears the user's access token from the Redux store.

    The slice also defines a selector function, selectCurrentToken, which can be used to get the current user's access token from the Redux store.

  • authApiSlice.jsx - It is used for authentication. It injects three endpoints into the existing API slice:
    • login: This endpoint allows users to log in to the application. It takes the user's credentials as input and returns an access token if the credentials are valid.
    • sendLogout: This endpoint logs the user out of the application. It invalidates the user's access token and removes it from the browser's local storage.
    • refresh: This endpoint refreshes the user's access token. It takes the user's refresh token as input and returns a new access token if the refresh token is valid.

    The API slice also exports three hooks that can be used to interact with the endpoints:

    • useLoginMutation: This hook can be used to log in to the user.
    • useSendLogoutMutation: This hook can be used to log the user out.
    • useRefreshMutation: This hook can be used to refresh the user's access token.
  • Store.jsx - It configures a Redux Toolkit store with the apiSlice API slice and the authReducer reducer. It also adds the apiSlice.middleware middleware to the store and set up listeners for query updates.
  • apiSlice.jsx - The apiSlice function creates a new Redux Toolkit API slice using the baseQueryWithReauth function as the base query. This means that all of the endpoints defined in the apiSlice function will automatically re-authenticate the user

Login Components

  • Login.jsx - It’s responsible for allowing users to log into your application. It does this by using the useLoginMutation hook to call the login mutation, which will authenticate the user and generate an access token if the user's credentials are valid.
  • PersistsLogin.jsx - This in your code is responsible for transparently refreshing the user's access token and keeping them logged in, even if their token expires. It does this by using the useRefreshMutation hook to call the refresh mutation, which will generate a new access token if the user's refresh token is still valid.

    The component also uses the usePersist hook to determine whether the user has enabled persistent login. If they have, and their access token has expired, the component will automatically call the refresh mutation to obtain a new access token.

  • Prefetch.jsx - It is responsible for prefetching the list of users from the user API. It does this by using the usersApiSlice.util.prefetch function, which dispatches a prefetch action to the Redux store. The prefetch action will then be handled by the usersApiSlice, which will make a request to the user API and cache the results in the Redux store.
